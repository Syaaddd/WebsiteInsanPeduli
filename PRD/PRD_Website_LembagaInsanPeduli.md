## 1. Ringkasan Eksekutif

Website ini merupakan kehadiran digital resmi **Lembaga Insan Peduli (LIP)**, sebuah yayasan sosial kemanusiaan yang bergerak di bidang sosial, pendidikan, kesehatan jiwa, dan pemberdayaan masyarakat. Website dibangun secara bertahap — dimulai dari versi MVP (Minimum Viable Product) yang menampilkan identitas lembaga dan informasi program, kemudian dikembangkan lebih lanjut sesuai kebutuhan operasional yang berjalan.

> Pendekatan ini dipilih karena lembaga masih dalam tahap awal berdiri (2026), sehingga website dibuat **"ada dulu"** sebagai fondasi digital, lalu diperkaya fitur seiring berjalannya kegiatan.

## 3. Tujuan Produk

### 3.1 Tujuan Utama
1. Membangun **kehadiran digital resmi** LIP agar mudah ditemukan masyarakat dan calon donatur.
2. Menyampaikan **profil, visi, misi, dan program kerja** lembaga secara jelas dan terpercaya.
3. Membuka **jalur komunikasi** antara lembaga dengan masyarakat, donatur, dan mitra.
4. Membangun **kepercayaan publik** melalui tampilan yang profesional dan transparan.

### 3.2 Tujuan Jangka Panjang (Fase Berikutnya)
- Menerima donasi online secara langsung melalui website.
- Menampilkan laporan kegiatan dan keuangan secara berkala.
- Membangun komunitas donatur dan relawan.

## 4. Target Pengguna (User Persona)

### Persona 1 — Calon Donatur
- **Siapa:** Masyarakat umum, pengusaha, atau instansi yang ingin berdonasi.
- **Kebutuhan:** Ingin tahu profil lembaga, program yang dijalankan, dan cara berdonasi.
- **Pain point:** Ragu jika lembaga tidak punya website resmi dan terkesan tidak profesional.

### Persona 2 — Penerima Manfaat / Masyarakat Umum
- **Siapa:** Warga yang membutuhkan bantuan atau ingin mendaftar sebagai penerima program.
- **Kebutuhan:** Cari informasi cara mendapatkan bantuan dan kontak lembaga.

### Persona 3 — Mitra & Instansi
- **Siapa:** Pemerintah desa, NGO lain, perusahaan (CSR), media.
- **Kebutuhan:** Verifikasi legalitas dan program lembaga sebelum berkolaborasi.

## 6. Detail Program Kerja (Konten Utama)

Konten berikut diambil dari proposal resmi LIP dan menjadi isi utama halaman **Program Kerja**:

### 6.1 Program Peduli ODGJ ⭐ (Program Unggulan)
Program kemanusiaan untuk membebaskan dan memuliakan Orang Dengan Gangguan Jiwa (ODGJ) dari pemasungan, penelantaran, dan stigma sosial. Meliputi identifikasi, rujukan medis, rumah singgah, rehabilitasi, dan reintegrasi sosial.

### 6.2 Safari Sosial (Safari Peduli)
Kunjungan langsung ke desa terpencil dan wilayah membutuhkan. Membawa paket sembako, santunan yatim, perlengkapan ibadah, dan tausiyah. Dilaksanakan 3× per bulan (rutin) + setiap minggu saat Ramadhan.

### 6.3 Program Santunan
- Santunan anak yatim & piatu (bulanan + beasiswa pendidikan)
- Santunan janda dhuafa (uang tunai / sembako + pemberdayaan)
- Santunan lansia (kunjungan rutin + paket sembako + pendampingan rohani)

### 6.4 Kemanusiaan & Bantuan Bencana
Distribusi logistik darurat, perbaikan rumah korban bencana, trauma healing, dan posko tanggap darurat yang siap diaktifkan sewaktu-waktu.

### 6.5 Pendidikan & Pemberdayaan
Beasiswa SD–SMA, Taman Baca Masyarakat (TBM), bimbingan belajar gratis, dan program wirausaha sosial bagi kaum dhuafa usia produktif.

### 6.6 Program Keagamaan
Kajian rutin, kegiatan Ramadhan (buka puasa, sahur on the road, zakat fitrah), program Qurban, pembagian Al-Quran, dan pembinaan tahfidz anak-anak.

## 7. Spesifikasi Teknis

### 7.1 Rekomendasi Teknologi (Fase 1)

| Aspek | Pilihan | Alasan |
|---|---|---|
| **Bahasa** | HTML + CSS + JavaScript (Vanilla) | Paling simpel, tidak butuh server khusus |
| **Styling** | CSS murni / Tailwind CSS | Cepat dan ringan |
| **Hosting** | Niagahoster / IDCloudHost / Netlify | Terjangkau, support Indonesia |
| **Domain** | `.org` atau `.id` | Cocok untuk yayasan/lembaga |
| **CMS** | Tidak ada di Fase 1 | Konten statis, di-hardcode di HTML |

### 7.2 Rekomendasi Teknologi (Fase 2)

| Aspek | Pilihan | Alasan |
|---|---|---|
| **Platform** | WordPress + Elementor | Mudah kelola konten tanpa coding |
| **Payment** | Midtrans / Xendit | Payment gateway lokal terpercaya |
| **Database** | MySQL (via WordPress) | Sudah terintegrasi |

### 7.3 Performa & Aksesibilitas
- Waktu load halaman: < 3 detik pada koneksi 4G
- Responsif di mobile, tablet, dan desktop
- Warna kontras cukup (WCAG AA minimum)
- Gambar dikompresi sebelum upload

## 10. Kriteria Keberhasilan (Success Metrics)

### Fase 1 — MVP
- [ ] Website live dan dapat diakses publik
- [ ] Semua 4 halaman terisi konten lengkap
- [ ] Tampil baik di HP (mobile-friendly)
- [ ] Informasi kontak (WhatsApp/telepon) berfungsi
- [ ] Load time < 3 detik

### Fase 2 — Pengembangan
- [ ] Rata-rata kunjungan website ≥ 100 pengunjung/bulan
- [ ] Minimal 1 donasi online masuk per bulan via website
- [ ] Laporan kegiatan diupdate minimal 1× per bulan
- [ ] Formulir relawan/bantuan berfungsi dan terkoneksi ke admin

---

## 11. Risiko & Mitigasi

| Risiko | Dampak | Mitigasi |
|---|---|---|
| Konten belum lengkap saat launch | Halaman kosong / kurang profesional | Gunakan placeholder sementara, isi bertahap |
| Tidak ada yang update konten | Website stagnan | Tunjuk admin konten dari pengurus (misal: Sekretaris) |
| Domain/hosting tidak diperpanjang | Website offline | Catat jadwal renewal, bayar minimal 1 tahun di muka |
| Gambar/foto kegiatan belum ada | Tampilan kurang menarik | Gunakan ilustrasi SVG atau stock photo gratis sementara |
